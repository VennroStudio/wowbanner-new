# Frontend API Docs — Документация для фронтенда

При создании каждого нового Action — добавить описание эндпоинта в `docs/frontend/{Module}.md`.

- Один файл на модуль; новый модуль — новый файл по шаблону ниже
- Существующий модуль — дописать метод в конец файла

---

## Шаблон файла

```markdown
# Фронт: {Module}

Краткое описание модуля и его назначения.

---

## Базовые вещи

- **Base URL:** `{BACKEND_BASE_URL}/v1`
- **Content-Type:** `application/json` для запросов с телом
- **Успех с данными:** `{ "data": { ... } }`
- **Ошибка домена (409):** `{ "error": { "code": number, "message": string } }`
- **Ошибка валидации (422):** `{ "validations": [{ "field": string, "message": string }] }`
- **401** — невалидный/отсутствующий JWT

---

## 1. {Название} ({ActionClass})

**Цель:** что делает эндпоинт.

**Запрос:**
- Метод: `POST` / `GET` / `PATCH` / `DELETE`
- URL: `/v1/{path}`
- Заголовок (если защищённый): `Authorization: Bearer <access_token>`

\```json
{
  "field": "value"
}
\```

**Валидация полей:**
- `field` — описание ограничений

**Успех ({status_code}):**
\```json
{ "data": { ... } }
\```

**Ошибки:**
- `409` `error.code` — описание
- `422` — ошибки валидации
- `401` — требуется авторизация

**На фронте:** описание поведения UI после успешного ответа.
```
