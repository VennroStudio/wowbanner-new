# Фронт: Client

Модуль управления клиентской базой. Позволяет создавать клиентов (физических и юридических лиц), привязывать к ним несколько номеров телефонов и компаний, а также выполнять поиск и фильтрацию.

---

## Базовые вещи

- **Base URL:** `{BACKEND_BASE_URL}/v1`
- **Content-Type:** `application/json` для запросов с телом
- **Успех с данными:** `{ "data": { ... } }`
- **Ошибка домена (409):** `{ "error": { "code": number, "message": string } }`
- **Ошибка валидации (422):** `{ "validations": [{ "field": string, "message": string }] }`
- **401** — невалидный/отсутствующий JWT

---

## 1. Список клиентов (GetClientsAction)

**Цель:** Получение списка клиентов с пагинацией и поиском.

**Запрос:**
- Метод: `GET`
- URL: `/v1/clients`
- Заголовок: `Authorization: Bearer <access_token>`
- Query параметры:
    - `page` — номер страницы (default: 1)
    - `perPage` — кол-во на странице (default: 20)
    - `search` — строка поиска (по ФИО, email, телефону или названию компании)

**Успех (200):**
```json
{
  "data": {
    "count": 125,
    "items": [
      {
        "id": 1,
        "last_name": "Иванов",
        "first_name": "Иван",
        "middle_name": "Иванович",
        "email": "ivanov@example.com",
        "type": { "id": 1, "label": "Физическое лицо" },
        "docs": { "id": 1, "label": "Паспорт" },
        "phones": [
          { "id": 1, "type": { "id": 1, "label": "Мобильный" }, "phone": "+79991234567" }
        ],
        "companies": []
      }
    ]
  }
}
```

## 3. Создание клиента (CreateClientAction)

**Цель:** Регистрация нового клиента в системе.

**Запрос:**
- Метод: `POST`
- URL: `/v1/clients/create`
- Заголовок: `Authorization: Bearer <access_token>`

```json
{
  "lastName": "Петров",
  "firstName": "Петр",
  "middleName": "Петрович",
  "email": "petrov@example.com",
  "docs": 1,
  "type": 2,
  "info": "Новый клиент",
  "phones": [
    { "type": 1, "phone": "+79001112233" }
  ],
  "companies": [
    { "name": "ООО Вектор" }
  ]
}
```

**Валидация полей:**
- `lastName`, `firstName` — обязательные строки.
- `docs` — тип документа (ID из справочника).
- `type` — тип клиента (1 - Individual, 2 - Legal).
- `phones` — массив объектов `{type, phone}`.
- `companies` — массив объектов `{name}` (обязателен для юридических лиц).

**Успех (201):**
```json
{ "data": 1 }
```

**Ошибки:**
- `409` код `4` — Email уже занят.
- `409` код `5` — Номер телефона уже занят.
- `409` код `6` — Для юридического лица обязательна хотя бы одна компания.

## 4. Обновление клиента (UpdateClientAction)

**Цель:** Изменение данных существующего клиента и синхронизация контактов.

**Запрос:**
- Метод: `PATCH`
- URL: `/v1/clients/update/{id}`
- Заголовок: `Authorization: Bearer <access_token>`

```json
{
  "lastName": "Петров",
  "firstName": "Петр",
  "phones": [
    { "id": 1, "type": 1, "phone": "+79001112233" },
    { "type": 1, "phone": "+79008889900" }
  ],
  "companies": [
    { "id": 5, "name": "ООО Вектор Плюс" }
  ]
}
```

**Синхронизация:**
- Если в массиве `phones` или `companies` передан `id`, запись обновляется.
- Если `id` нет — создается новая.
- Записи, которые были в БД, но отсутствуют в запросе — будут удалены.

**Успех (200):**
```json
{ "data": 1 }
```

---

## 5. Удаление клиента (DeleteClientAction)

**Цель:** Полное удаление клиента из системы.

**Запрос:**
- Метод: `DELETE`
- URL: `/v1/clients/delete/{id}`
- Заголовок: `Authorization: Bearer <access_token>`

**Успех (200):**
```json
{ "data": 1 }
```

**На фронте:** После успешного удаления необходимо перенаправить пользователя на список клиентов и показать уведомление.
