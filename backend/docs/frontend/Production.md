# Фронт: Production (производство)

Сущность **производство**: поле `name` и вложенные коллекции **materials** и **prints** (структура полей — в примерах ответов ниже). Список и карточка доступны любому пользователю с валидным JWT; **создание, изменение и удаление** — только роли **администратор**.

---

## Базовые вещи

- **Base URL:** `{BACKEND_BASE_URL}/v1`
- **Content-Type:** `application/json` для запросов с телом
- **Защита:** все эндпоинты требуют заголовок `Authorization: Bearer <access_token>`
- **Успех с данными:** `{ "data": { ... } }`
- **Успех без полезной нагрузки (create/update/delete):** `{ "data": { "success": 1 } }`
- **Ошибка домена:** `{ "error": { "code": number, "message": string } }` (часто `409`)
- **Ошибка валидации (422):** `{ "validations": [{ "field": string, "message": string }] }`
- **401** — невалидный/отсутствующий JWT
- **403** — нет прав на операцию (не администратор при create/update/delete)

---

## 1. Список производств (GetProductionsAction)

**Цель:** пагинированный список с поиском **по названию**; в каждой строке — `id`, `name`, а также вложенные `materials` и `prints`.

**Запрос:**

- Метод: `GET`
- URL: `/v1/productions`
- Заголовок: `Authorization: Bearer <access_token>`

**Query-параметры:**

- `page` — номер страницы (по умолчанию на бэкенде: `1`)
- `perPage` — размер страницы (по умолчанию: `20`, максимум: `100`)
- `search` — подстрока в поле `name` (опционально)

**Успех (200):**

```json
{
  "data": {
    "count": 3,
    "items": [
      {
        "id": 1,
        "name": "Цех широкоформата",
        "materials": [
          {
            "id": 10,
            "material_option_id": 5
          }
        ],
        "prints": [
          {
            "id": 20,
            "print_id": 2
          }
        ]
      }
    ]
  }
}
```

**Ошибки:**

- `401` — требуется авторизация
- `422` — ошибки валидации query

**На фронте:** таблица/список с пагинацией и поиском.

## 2. Производство по ID (GetProductionByIdAction)

**Цель:** одна запись для карточки/редактирования с полными списками `materials` и `prints`.

**Запрос:**

- Метод: `GET`
- URL: `/v1/productions/{id}`
- Заголовок: `Authorization: Bearer <access_token>`

**Успех (200):**

```json
{
  "data": {
    "id": 1,
    "name": "Цех широкоформата",
    "materials": [
      {
        "id": 10,
        "material_option_id": 5
      }
    ],
    "prints": [
      {
        "id": 20,
        "print_id": 2
      }
    ]
  }
}
```

**Ошибки:**

- `401` — требуется авторизация
- `404` / доменная ошибка — производство не найдено

**На фронте:** при 404 показать «не найдено».

## 3. Создание производства (CreateProductionAction)

**Цель:** создать производство и при необходимости сразу заполнить массивы `materials` и `prints`.

**Запрос:**

- Метод: `POST`
- URL: `/v1/productions/create`
- Заголовок: `Authorization: Bearer <access_token>`

```json
{
  "name": "Цех широкоформата",
  "materials": [
    { "materialOptionId": 5 }
  ],
  "prints": [
    { "printId": 2 }
  ]
}
```

**Валидация полей:**

- `name` — обязательно, длина 2–255 символов
- `materials` — опционально; элементы: `{ "id"?: number, "materialOptionId": number }` (для создания `id` не передаётся)
- `prints` — опционально; элементы: `{ "id"?: number, "printId": number }`

**Успех (201):**

```json
{
  "data": {
    "success": 1
  }
}
```

**Ошибки:**

- `401`, `403`
- `422` — валидация

**На фронте:** после успеха обновить список; идентификатор нового производства в теле ответа не возвращается — при необходимости запросить список заново или открыть поиск по имени.

---

## 4. Обновление производства (UpdateProductionAction)

**Цель:** изменить название и **синхронизировать** коллекции `materials` и `prints` с переданными массивами.

**Запрос:**

- Метод: `PATCH`
- URL: `/v1/productions/update/{id}`
- Заголовок: `Authorization: Bearer <access_token>`

```json
{
  "name": "Цех №1 обновлённый",
  "materials": [
    { "id": 10, "materialOptionId": 5 },
    { "materialOptionId": 7 }
  ],
  "prints": [
    { "id": 20, "printId": 2 },
    { "printId": 3 }
  ]
}
```

**Синхронизация коллекций:**

- Если у элемента указан `id` и такая строка есть у этого производства — выполняется обновление.
- Если `id` нет — создаётся новая строка.
- Элементы, которые уже были у этого производства, но **отсутствуют** в переданном массиве — **удаляются**.

**Успех (200):**

```json
{
  "data": {
    "success": 1
  }
}
```

**Ошибки:**

- `401`, `403`
- `404` — производство не найдено
- `422` — валидация

**На фронте:** после успеха обновить карточку и список; при редактировании формы всегда отправлять полные актуальные массивы `materials` / `prints`, если ожидается полная синхронизация.

---

## 5. Удаление производства (DeleteProductionAction)

**Цель:** удалить производство вместе с его коллекциями `materials` и `prints`.

**Запрос:**

- Метод: `DELETE`
- URL: `/v1/productions/delete/{id}`
- Заголовок: `Authorization: Bearer <access_token>`
- Тело: нет

**Успех (200):**

```json
{
  "data": {
    "success": 1
  }
}
```

**Ошибки:**

- `401`, `403`
- `404` / доменная ошибка — производство не найдено

**На фронте:** убрать из списка или перенаправить на список после подтверждения.
