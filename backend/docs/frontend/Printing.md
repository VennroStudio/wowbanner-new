# Фронт: Printing (печать)

Справочник видов/типов печати: CRUD для админки. Все эндпоинты защищены JWT; операции разрешены роли **администратор** (см. бизнес-правила на бэкенде).

---

## Базовые вещи

- **Base URL:** `{BACKEND_BASE_URL}/v1`
- **Content-Type:** `application/json` для запросов с телом
- **Защита:** заголовок `Authorization: Bearer <access_token>`
- **Успех с данными:** `{ "data": { ... } }`
- **Успех без полезной нагрузки (create/update/delete):** `{ "data": { "success": 1 } }`
- **Ошибка домена:** `{ "error": { "code": number, "message": string } }` (часто `409`)
- **Ошибка валидации (422):** `{ "validations": [{ "field": string, "message": string }] }`
- **401** — невалидный/отсутствующий JWT
- **403** — нет прав на операцию

---

## 1. Список печати (GetPrintingsAction)

**Цель:** пагинированный список с поиском **по названию**.

**Запрос:**
- Метод: `GET`
- URL: `/v1/printings`
- Заголовок: `Authorization: Bearer <access_token>`

**Query-параметры:**
- `page` — номер страницы (по умолчанию на бэкенде: `1`)
- `perPage` — размер страницы (по умолчанию: `20`, максимум: `100`)
- `search` — подстрока в поле `name` (опционально)

**Успех (200):**

```json
{
  "data": {
    "count": 15,
    "items": [
      {
        "id": 1,
        "name": "УФ-печать"
      }
    ]
  }
}
```

**Ошибки:**
- `401` — требуется авторизация
- `422` — ошибки валидации query

**На фронте:** таблица/список с пагинацией и поиском.

---

## 2. Печать по ID (GetPrintingByIdAction)

**Цель:** одна запись для карточки/редактирования.

**Запрос:**
- Метод: `GET`
- URL: `/v1/printings/{id}`
- Заголовок: `Authorization: Bearer <access_token>`

**Успех (200):**

```json
{
  "data": {
    "id": 1,
    "name": "УФ-печать"
  }
}
```

**Ошибки:**
- `401` — требуется авторизация
- `404` / доменная ошибка — запись не найдена

**На фронте:** при 404 — «не найдено».

---

## 3. Создание записи печати (CreatePrintingAction)

**Цель:** добавить запись в справочник.

**Запрос:**
- Метод: `POST`
- URL: `/v1/printings/create`
- Заголовок: `Authorization: Bearer <access_token>`

```json
{
  "name": "УФ-печать"
}
```

**Валидация полей:**
- `name` — обязательно, длина 2–255 символов

**Успех (201):**

```json
{
  "data": {
    "success": 1
  }
}
```

**Ошибки:**
- `401`, `403`
- `422` — валидация

**На фронте:** после успеха обновить список или открыть деталь (если бэкенд начнёт возвращать id — смотреть OpenAPI).

---

## 4. Обновление записи печати (UpdatePrintingAction)

**Цель:** изменить название.

**Запрос:**
- Метод: `PATCH`
- URL: `/v1/printings/update/{id}`
- Заголовок: `Authorization: Bearer <access_token>`

```json
{
  "name": "Широкоформатная"
}
```

**Валидация:** как у создания для `name`.

**Успех (200):**

```json
{
  "data": {
    "success": 1
  }
}
```

**Ошибки:**
- `401`, `403`
- `404` — запись не найдена
- `422` — валидация

**На фронте:** обновить деталь и список.

---

## 5. Удаление записи печати (DeletePrintingAction)

**Цель:** удалить запись из справочника.

**Запрос:**
- Метод: `DELETE`
- URL: `/v1/printings/delete/{id}`
- Заголовок: `Authorization: Bearer <access_token>`
- Тело: нет

**Успех (200):**

```json
{
  "data": {
    "success": 1
  }
}
```

**Ошибки:**
- `401`, `403`
- `404` — запись не найдена

**На фронте:** убрать из списка или редирект после подтверждения.
